## 那些神秘的".dat.jpg"文件

如果你好奇地解压过艾诺迪亚4的APK，在`assets/common/game_res/`目录下会看到一堆扩展名为`.dat.jpg`的文件：

```
game.dat.jpg
m0.dat.jpg
m1.dat.jpg
m2.dat.jpg
...
m415.dat.jpg
i_tile.dat.jpg
i_mapfeature.dat.jpg
i_worldmap.dat.jpg
memorytext_zhhans.dat.jpg
```

**一眼看过去是".jpg"，实际上根本不是图片。**

这类文件其实是游戏的**资源包**，里面装着地图、精灵图、文本等核心数据。游戏运行时再把它们解析成内存中的对象。

---

## data.jpg 到底是什么格式？

以`game.dat.jpg`为例，文件结构大概是这个样子：

```
[外层容器] → [内层数据] → [具体资源]
```

### 第一层：外层容器（LZMA压缩）

文件开头通常是这样的字节序列：`01 00 5D 00`

这是一种**LZMA压缩**的标识。说人话就是——这段数据被压缩过，需要解压缩才能看到里面的内容。

> 为什么要压缩？
> 当然是为了省空间。一个地图文件原始大小可能有几MB，压缩后能减少一半以上。

### 第二层：内层结构（SNASYS格式）

解压之后，里面的数据也不是直接可读的，而是一种叫**SNASYS**的结构：

```
[Offset表] → [Entry块1] → [Entry块2] → ... → [Entry块N]
```

Offset表记录了每个Entry块在哪里；Entry块里才是真正地图数据、tile数据或者文本数据。

### 第三层：具体资源

不同文件装着不同东西：

- `m*.dat.jpg` → 416张游戏地图（m0到m415）
- `i_tile.dat.jpg` → 地图瓦片（Tile）图集
- `i_mapfeature.dat.jpg` → 地图装饰物/建筑（Feature）
- `i_worldmap.dat.jpg` → 世界地图大图
- `memorytext_zhhans.dat.jpg` → 中文文本（对话、说明等）

## 地图到底长什么样？

这是大家最关心的部分。解析完成后，单张地图的数据大概长这样：

**【图片占位：单张地图解析效果预览图】**
- 类型：PNG截图
- 建议尺寸：800x600 或更大
- 内容：展示一张解析后的游戏地图，带有底层的地面tiles、上层的装饰物、阴影等

每一张地图实际上由**多层**构成：

1. **Base Layer（基础层）** - 地面/草地/地板
2. **Layer（遮挡层）** - 桥、洞穴顶等半遮挡元素
3. **Shadow1 / Shadow2（阴影层）** - 树木、建筑的阴影
4. **Top（顶层）** - 旗帜、树冠等在角色头顶的元素
5. **Static Feature（静态装饰）** - 房屋、石头、建筑等

**【图片占位：地图图层分解示意图】**
- 类型：PNG截图
- 建议尺寸：1200x400
- 内容：将地图的各个图层拆开展示，Base→Layer→Shadow1→Shadow2→Top 依次排列

这种分层设计是为了实现**透视效果**——你走过桥洞时，桥会"挡"在你前面；你躲在树下时，树冠会"盖"在你头顶。

## Tile（瓦片）系统

**【图片占位：Tile Atlas拼贴图】**
- 类型：PNG截图
- 建议尺寸：1024x1024
- 内容：展示i_tile.dat.jpg解析后的瓦片图集，类似精灵图（sprite sheet）

游戏的地图不是一整张图片，而是用**瓦片（Tile）拼出来**的。

类似于我的世界（Minecraft）或者早期RPG Maker的方式——用几十上百个基础瓦片，通过不同的排列组合，就能构建出千变万化的地图场景。

每个瓦片有自己的：
- 宽度/高度
- 调色板（Palette）编号
- 偏移量（决定绘制位置）
- 是否可翻转（水平翻转、垂直翻转）

这样一套下来，416张地图使用的瓦片总数其实并不需要416个，而只需要**几十个瓦片set**就能覆盖所有地图。

## Feature（装饰物）系统

**【图片占位：Feature装饰物图集】**
- 类型：PNG截图
- 建议尺寸：1024x512
- 内容：展示i_mapfeature.dat.jpg解析后的装饰物图集

地图上那些房屋、树木、石头、墓碑、帐篷之类的东西，叫做**Feature（功能装饰）**。

它们跟Tile的区别是：
- Tile 通常是小块的（16x16 或 32x32）
- Feature 通常是大块的，可以有复杂的形状和多个绘制阶段

Features同样使用Atlas（拼图）的方式存储，每个Feature有：
- 锚点位置
- 宽高
- 所属调色板
- 绘制层级

**【图片占位：Feature在地图上的应用效果】**
- 类型：PNG截图
- 建议尺寸：600x400
- 内容：展示同一张地图上有Feature和没有Feature的对比

## 世界地图

**【图片占位：世界地图整体效果】**
- 类型：PNG截图
- 建议尺寸：1600x900 或更大
- 内容：展示完整的i_worldmap.dat.jpg解析结果，标注主要区域

`i_worldmap.dat.jpg` 包含了游戏的**世界地图**。

这张大图由多个sprite（精灵图）拼合而成，每个区域对应着不同的游戏地图。

游戏中的"世界地图"界面就是这张图，玩家点击各个区域就能进入对应的地图。

主要区域一览：

- 艾伦尼城：4张地图
- 哈肯沙漠：4张地图
- 哥布林巢穴：13张地图
- 帝国首都：15张地图
- 永恒之森：16张地图
- （更多区域省略...）

## 文本资源：memorytext

**【图片占位：文本解析界面截图】**
- 类型：PNG截图
- 建议尺寸：800x600
- 内容：展示memorytext_zhhans.dat.jpg的解析结果，包含text_id和对应文本

`memorytext_zhhans.dat.jpg` 装着游戏里所有的**中文文本**：
- NPC对话
- 任务说明
- 物品描述
- 系统提示

文本数据通过一个**偏移量表（Offset Table）**来索引，每个文本条目有一个唯一的`text_id`，游戏通过这个ID来查找对应的文本内容。

> 原文韩语文本存放在`memorytext_ko.dat.jpg`，中文翻译版存放在`memorytext_zhhans.dat.jpg`。
